# Contributing to Blockchain Swap

We're excited that you're interested in contributing to Blockchain Swap! This project is a community-driven Decentralized Exchange on the Blockchain network.

## 🤝 Code of Conduct
Please be respectful and professional in all interactions. We follow standard open-source diversity and inclusion guidelines.

## 🚀 How to Contribute

### 1. Reporting Bugs
- Use the GitHub Issue tracker.
- Describe the bug precisely and include steps to reproduce.
- Label the issue as `bug`.

### 2. Feature Requests
- Open an issue titled `[Feature] Description`.
- Describe why this feature is needed and its potential impact.

### 3. Submitting Pull Requests
- Fork the repository.
- Create a feature branch: `git checkout -b feature/your-feature-name`.
- Ensure all tests pass: `cargo test` and `npm run build`.
- Submit the PR into the `main` branch.

## 📝 Commit Message Conventions

We use standardized prefixes to ensure a clean and readable project history:
- `feat:` A new feature for the user.
- `fix:` A bug fix.
- `docs:` Documentation-only changes.
- `style:` Changes that do not affect the meaning of the code (white-space, formatting).
- `refactor:` A code change that neither fixes a bug nor adds a feature.
- `test:` Adding missing tests or correcting existing tests.
- `chore:` Changes to the build process or auxiliary tools and libraries.

**Example:** `feat: add slippage protection to router contract`

## 🛠️ Development Environment
- **Backend**: Rust (nightly/stable), Wasm32 target, Soroban-CLI.
- **Frontend**: Node.js 18+, NPM/PNPM.

Thank you for contributing to the future of Stellar DeFi!
